# UI Designer Skill

## Overview

Use this skill when the user wants a UI that looks production-ready, premium, and intentionally designed rather than generic or template-like.

## Core Direction

- Build for modern SaaS, enterprise dashboards, AI products, and national-scale platforms.
- Favor Apple HIG, Material Design 3, Stripe, Linear, Vercel, Notion, and Figma Community quality.
- Keep the result Thai-first when language, hierarchy, or content should feel local and natural.

## Non-Negotiables

- Use `shadcn/ui`, `Tailwind CSS`, `lucide-react`, and subtle `Framer Motion`.
- Use an 8px spacing system, generous white space, premium cards, soft shadows, and rounded corners.
- Make layouts responsive and accessible by default.
- Keep the color system consistent and restrained.
- Avoid Bootstrap-like, government-portal-like, tiny-button, plain-table, or generic Tailwind demo aesthetics.

## Layout Expectations

For landing pages, prefer this structure unless the user asks otherwise:

1. Hero
2. Problem
3. Solution
4. Workflow
5. Features
6. Screenshots
7. CTA
8. Footer

## Design Rules

- Lead with strong typography and clear visual hierarchy.
- Use large spacing and calm compositions; do not crowd the page.
- Make primary actions obvious and secondary actions quiet.
- Use motion sparingly to add polish, not distraction.
- Make every screen feel finished, realistic, and ready to ship.

## Output Standard

When designing, deliver interfaces that look like a polished product launch, not an exercise or placeholder.
